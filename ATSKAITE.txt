1. SISTĒMAS TEHNOLOĢISKAIS NODROŠINĀJUMS

Izmantotās tehnoloģijas:
- PHP 7+ - serverī palaistās programmas valoda
- MySQL - datu bāze informācijas glabāšanai
- HTML/CSS - lietotāja saskarnes izveide
- JavaScript - dinamiske funkcionalitātes izveidošanai
- Apache - web serveris (XAMPP)

Rīki un programmas:
- XAMPP - lokāli serveris uz datora
- phpMyAdmin - datu bāzes pārvaldības rīks
- Git - versiōņu kontrole koda saglabāšanai
- GitHub - koda repozitorijs online

Sistēmas apraksts:
O! Pica ir vienkārša picu pasūtījuma sistēma. Lietotāji var 
pēc pieceļā skatīt picas, pievienot tās grozam, pieņemt. 
Administrators var redzēt pasūtījumus un pārvaldīt sistēmu.


2. DATU BĀZES PROJEKTĒŠANA

2.1. Konceptuālais līmenis
Bāzē glabājam: lietotājus, picas, pasūtījumus, sūtījuma vienības.
Lietotājs var veikt vairākus pasūtījumus.
Katrs pasūtījums satur vairākas picas.

2.2. Loģiskais līmenis
Tabulas:
- users (lietotāji)
- pizzas (picu katalogs)
- orders (pasūtījumi)
- order_items (pasūtījuma picas)
- support_requests (klientu ziņas)

2.3. Fiziskais līmenis
Izveidotas SQL tabulas ar laukiem un attiecībām.

2.4. Tabulu struktūras

USERS tabula:
- user_id (primārā atslēga)
- username (lietotāja vārds)
- email (e-pasts)
- password_hash (parole - šifrēta)
- Lauks ir obligāts un unikāls

PIZZAS tabula:
- pizza_id (primārā atslēga)
- name (picas nosaukums)
- price (cena)
- image (attēls)
- Katra pica ir unikāla

ORDERS tabula:
- order_id (primārā atslēga)
- user_id (ārējā atslēga uz users)
- order_date (pasūtījuma datums)
- total_price (kopējā cena)
- Saistīta ar lietotāju

ORDER_ITEMS tabula:
- order_item_id (primārā atslēga)
- order_id (ārējā atslēga uz orders)
- pizza_id (ārējā atslēga uz pizzas)
- quantity (daudzums)
- unit_price (cena par vienību)
- Saistīta ar pasūtījumu un picu

SUPPORT_REQUESTS tabula:
- id (primārā atslēga)
- name (vārds)
- email (e-pasts)
- message (ziņojums)
- timestamp (laiks)
- Klientu ziņas


3. ER DIAGRAMMA

Entītijas:
- USERS (lietotāji)
- PIZZAS (pical)
- ORDERS (pasūtījumi)
- ORDER_ITEMS (pasūtījuma lietas)
- SUPPORT_REQUESTS (palīdzības pieprasījumi)

Attiecības:
- USERS 1:N ORDERS (viens lietotājs - daudz pasūtījumu)
- ORDERS 1:N ORDER_ITEMS (viens pasūtījums - daudz vienu)
- PIZZAS 1:N ORDER_ITEMS (viena pica - daudz pasūtījumos)

Normalizācija:
- 1NF: Bez atkārtošanās grupu
- 2NF: Visi neklastera lauki ir atkarīgi no primārās atslēgas
- 3NF: Nav transitīvas atkarības

5. PROGRAMMAS FUNKCIONALITĀTES IZSTRĀDE

CRUD darbības:

CREATE (izveidošana):
- Jauns lietotājs - reģistrēšanās (register.php)
- Jauns pasūtījums - grozg apstiprināšana
- Jauna ziņa - contact forma (help.php)

READ (lasīšana):
- Picu saraksts no DB
- Lietotāja profils
- Pasūtījuma vēsture
- Admin - redzēt visus pasūtījumus

UPDATE (rediģēšana):
- Lietotāja dati (profile.php)
- Grozs pirms maksājuma

DELETE (dzēšana):
- Admin dzēš pasūtījumus
- Admin dzēš ziņas
- Lietotājs removē vienas picas no groza

SQL vaicājumi:

SELECT - datu lasīšana:
  SELECT * FROM pizzas;
  SELECT * FROM orders WHERE user_id = 5;

INSERT - jaunas rindas:
  INSERT INTO orders VALUES (null, 3, NOW(), 25.50);

UPDATE - datu mainīšana:
  UPDATE users SET email = 'jauns@mail.com' WHERE user_id = 1;

DELETE - datu dzēšana:
  DELETE FROM orders WHERE order_id = 10;

Datubāzes savienojums:
$conn = new mysqli('localhost', 'root', '', 'opica_db');


6. KĻŪDU MEKLĒŠANA UN NOVĒRTĒŠANA

SQL kļūdas:
- Sintakses kļūdas SQL vaicājumos
- Risinājums: Pārbaudītu MySQL logu phpMyAdmin
- Nokļūdīta tabula vai lauks nosaukums
- Risinājums: Pēc tabulām un laukiem phpMyAdmin

GUI kļūdas:
- Formas nepareizi apstrāda datus
- Risinājums: Pievienot kļūdu paziņojumus HTML
- Formas nevalidē ievadi
- Risinājums: Pievienot JavaScript pārbaudi

Risinājumi:
- isset() un empty() pārbaude ievadiem
- htmlspecialchars() drošībai pret XSS
- Regularās izteiksmes datu validācijai
- Try-catch kļūdu apstrādei
